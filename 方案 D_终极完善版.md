## 二、补充完善方案

### 2.1 技能间数据传递机制

**问题**：当多个 skills 协作时，如何传递数据？

**示例场景**：
```
用户："帮我写论文，需要查文献和数据分析"

涉及 skills：
1. literature-search → 产出：文献列表、阅读笔记
2. data-analysis → 产出：分析结果、图表
3. research-paper → 产出：论文

问题：如何让 research-paper 使用 literature-search 的产出？
```

**解决方案：工作空间共享区**

```
/workspace/
├── .claude/
│   └── skills/
└── .claude-workspace/
    └── current-session/
        ├── input/          # 用户输入
        ├── output/         # 各 skill 输出
        │   ├── literature-search/
        │   │   └── references.md
        │   ├── data-analysis/
        │   │   └── charts/
        │   └── research-paper/
        │       └── draft.md
        └── context.json    # 会话状态
```

**context.json 结构**：
```json
{
  "session_id": "20260430-001",
  "active_skills": [
    "literature-search",
    "data-analysis",
    "research-paper"
  ],
  "shared_data": {
    "references": ["文献 1", "文献 2"],
    "analysis_results": ["图表 1", "图表 2"],
    "topic": "环境会计"
  },
  "status": "in_progress"
}
```

### 2.2 技能版本管理

**问题**：skill 更新后，索引如何同步？

**解决方案：版本清单**

```markdown
# skills-version.md

## 版本记录

| 技能名 | 版本 | 更新日期 | 变更内容 | 状态 |
|--------|------|----------|----------|------|
| exam-prep-workflow | v1.2 | 2026-04-30 | 增加记忆卡片功能 | ✅ 活跃 |
| competition-prep-workflow | v1.1 | 2026-04-29 | 优化学创杯模板 | ✅ 活跃 |
| research-paper-isolated | v2.0 | 2026-04-30 | 双模式分离 | ✅ 活跃 |
| data-analysis-workflow | v1.0 | 2026-04-30 | 初始版本 | ✅ 活跃 |

## 索引同步规则

1. 每次更新 skill 后，必须更新 skills-version.md
2. 重大变更（vX.0）需要更新 skills-index.md 的描述
3. 废弃技能标记为 ⚠️ 已废弃，保留 30 天后删除
```

**更新检查工作流**：
```bash
# 更新 skill 后执行
./skills.sh check-version
# 输出：
# - 检查索引是否同步
# - 检查关键词映射是否更新
# - 检查 card 文件是否需要刷新
```

### 2.3 错误处理完整流程

**异常类型及处理**：

```markdown
## 异常 1：Skill 文件不存在

**触发**：索引指向的文件被删除或移动

**处理流程**：
1. 检测到文件不存在
2. 记录错误日志
3. 回退方案：
   - 选项 A：使用相似 skill 替代
   - 选项 B：询问用户是否手动指定
   - 选项 C：不触发 skill，正常对话
4. 修复索引

**响应示例**：
"抱歉，我找不到对应的技能文件。你是想：
A. 使用类似的 exam-prep-workflow
B. 详细描述你的需求，我直接帮你
C. 稍后处理（我会修复这个问题）"
```

```markdown
## 异常 2：意图识别置信度<20%

**触发**：完全无法理解用户意图

**处理流程**：
1. 不触发任何 skill
2. 通用回应 + 引导
3. 记录到日志，后续分析

**响应示例**：
"我有点没理解你的需求😅 你能详细说说吗？

或者直接告诉我你想：
- 学习/复习 → exam-prep
- 参加比赛 → competition-prep
- 写论文 → research-paper
- 处理数据 → data-analysis
- 其他需求 → 请描述"
```

```markdown
## 异常 3：Skill 执行失败

**触发**：skill 执行过程中出错

**处理流程**：
1. 捕获错误
2. 保存当前进度
3. 告知用户
4. 提供备选方案

**响应示例**：
"抱歉，执行过程中遇到问题：[错误信息]

已保存当前进度，你可以：
A. 重试一次（可能是临时问题）
B. 我换一种方式帮你
C. 手动完成剩余步骤

当前已完成：
✅ 知识点提取
✅ 思维导图生成
❌ 记忆卡片制作（失败）"
```

```markdown
## 异常 4：用户打断

**触发**：执行到一半用户改变需求

**处理流程**：
1. 检测意图变化（置信度突然下降）
2. 暂停当前 skill
3. 保存进度
4. 询问确认

**响应示例**：
"我注意到你可能想切换任务？

当前任务（exam-prep）进度：50%
- ✅ 已完成：知识点提取
- ⏳ 进行中：制作记忆卡片
- ⏸️ 待完成：制定复习计划

你是想：
A. 继续当前任务
B. 切换到新任务（新任务是什么？）
C. 先保存当前进度，以后继续"
```

### 2.4 性能优化方案

**问题**：频繁读取 skill 文件影响响应速度

**解决方案：多级缓存**

```
┌─────────────────────────────────┐
│ L1 缓存：内存缓存（热数据）       │
│ 最常用的 5 个 skills              │
│ 命中时间：<100ms                  │
│ 失效策略：LRU，30 分钟            │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│ L2 缓存：文件缓存（温数据）       │
│ 最近使用的 20 个 skills            │
│ 命中时间：<500ms                  │
│ 失效策略：24 小时                 │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│ L3 存储：完整 skills（冷数据）     │
│ 88 个技能文件                     │
│ 读取时间：1-2 秒                  │
│ 永久保存                          │
└─────────────────────────────────┘
```

**缓存预热策略**：
```markdown
## 主动预热

场景：检测到用户开始某个任务
操作：预加载相关 skills

示例：
用户说"我要准备考试"
→ 预热：exam-prep, weekly-planning, data-analysis（可能有数据分析作业）

## 被动预热

场景：用户确认使用某个 skill
操作：预判下一步可能用到的 skills

示例：
用户确认使用 exam-prep
→ 预热：weekly-planning（复习计划）、knowledge-manage（整理笔记）
```

### 2.5 个性化适配

**问题**：不同用户表达方式不同

**解决方案：用户画像学习**

```json
{
  "user_profile": {
    "user_id": "default",
    "preferred_skills": [
      "exam-prep-workflow",
      "competition-prep-workflow"
    ],
    "expression_style": {
      "direct": 0.8,
      "indirect": 0.2
    },
    "common_phrases": [
      "帮我弄一下",
      "怎么做",
      "需要准备什么"
    ],
    "threshold_adjustment": {
      "direct_trigger": 75,
      "confirm_trigger": 55,
      "ask_confirm": 35
    }
  }
}
```

**学习机制**：
```markdown
## 正向反馈

用户说"对，就是这个" → 强化当前匹配模式
用户完成任务 → 记录使用的 skill
用户重复使用 → 提高该 skill 的优先级

## 负向反馈

用户说"不对" → 降低当前匹配权重
用户中断任务 → 记录原因
用户手动指定 skill → 学习用户偏好

## 定期更新

每周回顾用户行为
调整个性化阈值
更新常用表达短语
```

### 2.6 可解释性增强

**问题**：用户不知道为什么触发某个 skill

**解决方案：透明化说明**

```markdown
## 触发时告知

"检测到你想【写论文】（关键词匹配），我将使用【research-paper-workflow】来帮你。

这个工作流包括：
1. 文献检索与整理
2. 内容提取与翻译
3. 格式化与配图
4. 导出 Word/PDF

预计需要 15-20 分钟，准备好了吗？"

## 低置信度解释

"我注意到你说【看看这个比赛】，我 60% 确定你是想准备参赛。

匹配依据：
✅ "比赛" → competition-prep
✅ "帮我" → 强意图信号
❌ "看看" → 弱化了意图

你是想准备参赛，还是了解比赛？"

## 不触发解释

"你提到【论文很难】，我理解这是对写论文的感叹，不是请求帮助。

如果你想让我帮忙写论文，可以说：
- "帮我写论文"
- "我需要写论文"
- 用/research-paper 命令

有其他我能帮的吗？"
```

### 2.8 技能生命周期管理

```markdown

- 正常使用

- 索引中正常显示

- 标记⚠️

### 已移除（Removed）
- 从索引中删除
- 文件归档
- 保留变更记录

## 状态流转

新建 → 测试 → 活跃 → 已废弃 → 已移除
         ↑                ↓
         └──── 更新 ──────┘
```

## 五、方案 D 终极版总结

### 完整架构

```
┌─────────────────────────────────────────────────┐
│           用户输入                                │
└────────────────────┬────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│  第 1 层：关键词粗筛（多级权重）                   │
│  - 一级词（100%）、二级词（70%）、三级词（50%）    │
│  - 组合增强、同义词扩展                          │
└────────────────────┬────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│  第 2 层：语义精筛                                │
│  - 否定词识别、意图强度、上下文理解              │
└────────────────────┬────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│  第 3 层：置信度评估                              │
│  - 计算最终置信度、决策（触发/确认/不触发）       │
└────────────────────┬────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│  第 4 层：用户确认（低置信度时）                   │
│  - 高置信度：直接执行                            │
│  - 中置信度：告知式确认                          │
│  - 低置信度：询问式确认                          │
└────────────────────┬────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│  执行 Skill                                      │
│  - 加载完整 skill 文件                             │
│  - 数据传递（多 skill 协作）                      │
│  - 进度保存、错误处理                            │
└─────────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│  清理与缓存                                      │
│  - 释放 skill 内容                                 │
│  - 更新缓存、记录日志                            │
└─────────────────────────────────────────────────┘
```
