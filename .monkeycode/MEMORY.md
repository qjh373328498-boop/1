# 用户指令记忆

本文件记录了用户的指令、偏好和教导，用于在未来的交互中提供参考。

## 格式

### 用户指令条目
用户指令条目应遵循以下格式：

[用户指令摘要]
- Date: [YYYY-MM-DD]
- Context: [提及的场景或时间]
- Instructions:
  - [用户教导或指示的内容，逐行描述]

### 项目知识条目
Agent 在任务执行过程中发现的条目应遵循以下格式：

[项目知识摘要]
- Date: [YYYY-MM-DD]
- Context: Agent 在执行 [具体任务描述] 时发现
- Category: [代码结构|代码模式|代码生成|构建方法|测试方法|依赖关系|环境配置]
- Instructions:
  - [具体的知识点，逐行描述]

## 去重策略
- 添加新条目前，检查是否存在相似或相同的指令
- 若发现重复，跳过新条目或与已有条目合并
- 合并时，更新上下文或日期信息
- 这有助于避免冗余条目，保持记忆文件整洁

- Date: 2026-04-30
- Context: 用户教导写论文时的知识库使用原则
- Instructions:
  - **隔离模式（默认）**：写论文时默认不参考知识库，仅使用外部权威文献和用户提供的资料
  - **知识库模式**：仅当用户明确说"参考知识库"、"用知识库的资料"、"结合我之前的资料"等时，才参考知识库

  - 适用场景区分：
    - 正式学术论文、学术竞赛 → 隔离模式
    - 课程作业、学习整理、比赛报告 → 知识库模式（需用户明确授权）
