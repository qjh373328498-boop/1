本文件记录了用户的指令、偏好和教导，用于在未来的交互中提供参考。

## 格式

### 用户指令条目
用户指令条目应遵循以下格式：

[用户指令摘要]
- Date: [YYYY-MM-DD]
- Context: [提及的场景或时间]
- Instructions:
  - [用户教导或指示的内容，逐行描述]

### 项目知识条目
Agent 在任务执行过程中发现的条目应遵循以下格式：

[项目知识摘要]
- Date: [YYYY-MM-DD]
- Context: Agent 在执行 [具体任务描述] 时发现
- Category: [代码结构|代码模式|代码生成|构建方法|测试方法|依赖关系|环境配置]
- Instructions:
  - [具体的知识点，逐行描述]

## 去重策略
- 添加新条目前，检查是否存在相似或相同的指令
- 若发现重复，跳过新条目或与已有条目合并
- 合并时，更新上下文或日期信息
- 这有助于避免冗余条目，保持记忆文件整洁

- Date: 2026-05-01
- Context: Agent 在执行德国心脏病桌游介绍视频生成任务时发现
- Category: 构建方法
- Instructions:
  - **本地视频生成能力**：
    - ✅ 优势：edge-tts 配音质量好、字幕同步准确、BGM 生成、全自动流程
    - ❌ 限制：画面单调（纯色背景 + 文字）、无真实素材、渲染慢（3 分钟视频需 30 分钟）、中文字体需手动安装

    2. 导入剪映/必剪等专业工具
    3. 添加真实素材（网络图片/实拍视频）
    4. 添加转场特效和装饰元素
    5. 导出最终视频
  - **用户期望管理**：当用户要求生成视频时，必须明确说明能力范围，推荐方案 B（混合制作）为最优解
  - **系统依赖**：中文字体必须预先安装 `apt-get install fonts-wqy-microhei fonts-wqy-zenhei`
