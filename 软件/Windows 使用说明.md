# Windows 使用说明

## ⚠️ 重要：先转换编码再使用！

由于这些`.bat` 文件是在 Linux 系统创建的（UTF-8 编码），在 Windows 上运行前**必须**转换为 ANSI 编码，否则会显示乱码。

---

## 🔧 转换步骤（必须执行）

### 方法 1：使用记事本（推荐）

1. 找到以下文件：
   - `软件/启动菜单.bat`
   - `软件/软件启动工具/一键启动-Windows 系统.bat`
   - `软件打包工具/一键打包-Windows.bat`

2. **对每个文件执行**：
   - 右键点击文件 → 打开方式 → 记事本
   - 点击菜单：**文件 → 另存为**
   - 在"编码"下拉框选择：**ANSI**
   - 点击"保存"，覆盖原文件
   - 确认替换

### 方法 2：批量转换（如果有多个文件）

1. 打开记事本
2. 复制下面内容保存为 `convert.bat`：

```batch
@echo off
chcp 936 >nul
for %%f in (*.bat) do (
    copy "%%f" "%%f.tmp"
    type "%%f.tmp" > "%%f"
    del "%%f.tmp"
)
echo 转换完成
pause
```

3. 把 `convert.bat` 放到包含 `.bat` 文件的文件夹
4. 双击运行 `convert.bat`

## 🚀 使用方法

### 启动软件

1. 双击 `软件/启动菜单.bat`
2. 输入数字选择要启动的软件
3. 软件会在新窗口启动

### 打包为 EXE

1. 在 Windows 上打开 `软件打包工具/` 文件夹
2. **先按上面步骤转换编码**
3. 双击运行 `一键打包-Windows.bat`
4. 等待打包完成（约 5-10 分钟）
5. 在 `dist` 文件夹找到 `.exe` 文件

## ❓ 常见问题

### Q: 为什么需要转换编码？
A: Linux 使用 UTF-8 编码，Windows CMD 使用 GB2312/ANSI 编码，不转换会显示乱码。

### Q: 转换后仍有乱码怎么办？
A: 检查是否真的保存为 ANSI 编码，重新执行转换步骤。

### Q: 可以直接运行吗？
A: 可以运行，但中文会显示为乱码。必须先转换为 ANSI 编码。

### Q: 转换编码后还是报错怎么办？
A: 可能是路径中有中文字符。尝试将整个文件夹移到英文路径下，如 `C:\Software\`

## ✅ 验证是否成功

转换成功后，双击 `启动菜单.bat` 应该看到：

```
====================================
启动菜单
====================================

1 - 启动第一个软件
2 - 启动第二个软件
3 - 退出

选择 (1-3):
```

如果看到乱码（如 `''`, `'鏃'` 等），说明编码转换失败，请重新执行转换步骤。

## 💡 最简单的方法

如果觉得转换编码太麻烦，可以直接：

1. **在 Windows 上重新创建 `.bat`文件**
2. 用记事本直接编写（默认就是 ANSI 编码）
3. 参考内容复制自现有文件

或者使用已经创建的英文版：
- `软件/run.bat`（英文界面，无需转换）
