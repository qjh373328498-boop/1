## ❌ 错误 1: 虚拟环境未创建或已损坏

### 错误信息
```
错误：虚拟环境未创建或已损坏
路径：G:\软件\财务工具箱\venv\Scripts\python.exe
```

### 原因
- 上次安装中断导致 venv 不完整
- 杀毒软件删除了 venv 中的文件
- 磁盘空间不足导致创建失败

### ✅ 解决方案

**方法 1：自动修复（推荐）**

直接重新运行启动脚本，会**自动检测并清理**损坏的虚拟环境：

```
双击：财务工具箱/一键启动.bat
或双击：软件/一键启动.bat → 选择 2

启动器会：
1. 检测到损坏的 venv
2. 自动删除旧的 venv
3. 重新创建新的 venv
4. 重新安装依赖
```

**方法 2：手动修复**

```cmd
# 1. 打开 CMD
# 2. 删除损坏的 venv
cd G:\软件\财务工具箱
rmdir /s /q venv

# 3. 重新运行启动脚本
双击：一键启动.bat
```

## ❌ 错误 2: 找不到 Python

### 错误信息
```
错误：找不到 Python
请确保已安装 Python 3.8+
```

### 原因
- 未安装 Python
- Python 未添加到 PATH

### ✅ 解决方案

**步骤 1: 安装 Python**

1. 访问 https://www.python.org/downloads/
2. 下载 Python 3.8 或更高版本
3. **重要**: 安装时勾选 ✅ "Add Python to PATH"

**步骤 2: 验证安装**

打开 CMD，输入：
```cmd
python --version
```

如果显示类似 `Python 3.11.5`，说明安装成功。

**步骤 3: 重新启动软件**

```
双击：一键启动.bat
```

## ❌ 错误 3: 路径乱码/No Python at '"/usr/bin\python.exe'

### 错误信息
```
Fatal error in launcher: Unable to create process using '"G:\??2\??\????"'
```

### 原因
- 路径中有中文字符
- Windows CMD 编码问题

### ✅ 解决方案

**方法 1: 使用独立启动脚本（最简单）**

```
直接双击各软件目录下的一键启动.bat

例如: 软件/财务工具箱/一键启动.bat
```

**方法 2: 移动到纯英文路径**

```
G:\软件\财务工具箱\  →  D:\Software\CW_Toolbox\
```

**方法 3: 已自动修复**

最新版的 `start.py` 已改进路径处理，使用主菜单也应该能正常工作了。

## ❌ 错误 4: pip 安装失败

### 错误信息
```
ERROR: Could not install packages due to an OSError: [Errno 13] Permission denied
```

### 原因
- 权限不足
- 文件被占用

### ✅ 解决方案

**方法 1: 以管理员身份运行**

1. 右键一键启动.bat
2. 选择"以管理员身份运行"

**方法 2: 关闭占用程序**

1. 关闭所有 CMD 窗口
2. 关闭正在运行的 Python 程序
3. 重新启动一键启动.bat

**方法 3: 手动删除 venv**

```cmd
# 删除 venv
cd G:\软件\财务工具箱
rmdir /s /q venv

# 重新创建
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 📋 预防建议

### 1. 使用纯英文路径
```
✅ D:\Software\财务工具箱\
✅ C:\Apps\CW_Toolbox\
❌ G:\软件\软件\财务工具箱\
```

### 2. 定期清理
```
# 每月清理一次 venv，保持健康
rmdir /s /q venv
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### 3. 关闭杀毒软件（临时）
某些杀毒软件会误报 Python 文件，安装时可以临时关闭。

### 4. 确保磁盘空间
至少预留 500MB 空间给虚拟环境和依赖包。
