# ⚠️ Windows 批处理文件编码重要说明

## 问题原因

Windows 的 CMD（命令提示符）默认使用 **GB2312/GBK 编码**（代码页 936），而不是 UTF-8。

如果 `.bat` 文件用 UTF-8 编码保存，运行时会显示**乱码**。

---

## ✅ 解决方案

### 方法 1：使用记事本转换编码（推荐）

1. 用**记事本**打开 `.bat` 文件
2. 点击 **文件 → 另存为**
3. 在"编码"下拉框选择 **ANSI**（不是 UTF-8）
4. 保存，覆盖原文件

### 方法 2：使用 VS Code 转换

1. 用 VS Code 打开 `.bat` 文件
2. 点击右下角的 **UTF-8** 编码显示
3. 选择 **Save with Encoding**
4. 选择 **Western (Windows 1252)** 或 **GB2312**
5. 保存

### 方法 3：使用 Notepad++

1. 用 Notepad++ 打开 `.bat` 文件
2. 点击菜单 **编码 → 转为 ANSI 编码**
3. 保存（Ctrl+S）

---

## 📋 需要转换的文件

所有包含中文的 `.bat` 文件都需要转换：

- `软件/启动菜单.bat`
- `软件打包工具/一键打包-Windows.bat`
- `软件/Windows 安装包/启动菜单.bat`

---

## 🔍 如何验证

转换后，双击运行 `.bat` 文件：

✅ **正确显示**：
```
学创杯 + 财务工具箱 - 启动菜单
请选择要启动的软件:
```

❌ **乱码显示**：
```
o'.
'欢.exe"'
'鏃犳晠鐨勯€夐」
```

---

## 💡 最佳实践

### 编写批处理文件时：

1. **避免使用 emoji 表情**（CMD 不支持）
2. **使用纯中文和 ASCII 字符**
3. **保存为 ANSI 编码**（不是 UTF-8）
4. **使用 `chcp 936`** 设置中文代码页

### 正确的示例：

```batch
@echo off
chcp 936 >nul
echo 正在启动程序...
```

### 错误的示例：

```batch
@echo off
chcp 65001 >nul
echo 🚀 正在启动...  REM ❌ emoji 会乱码
```

---

## 🆘 如果已经出现乱码

1. 关闭 CMD 窗口
2. 按照**方法 1** 用记事本另存为 ANSI 编码
3. 重新运行
