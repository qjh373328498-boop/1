# 乱码问题已修复

## 问题原因

Windows CMD 默认使用 **GB2312 编码**（代码页 936），而文件使用了 UTF-8 编码，导致中文显示为乱码。

## 已修复的文件

以下文件已更新为 **ANSI/GB2312 兼容格式**：

- `软件/启动菜单.bat`
- `软件/软件启动工具/一键启动-Windows 系统.bat`
- `软件打包工具/一键打包-Windows.bat`

## 修复内容

1. **移除所有 emoji 表情**（CMD 不支持）
2. **使用 `chcp 936`** 代替 `chcp 65001`
3. **简化输出格式**，使用纯中文和 ASCII 字符

## 使用方法

### 重要：保存为 ANSI 编码

在 Windows 上使用前，请用**记事本**打开每个 `.bat` 文件：

1. 右键 → 用记事本打开
2. 文件 → 另存为
3. 编码选择 **ANSI**（不是 UTF-8）
4. 保存

## 测试方法

1. 双击运行 `软件/启动菜单.bat`
2. 应该看到清晰的中文菜单
3. 输入数字选项测试功能

## 如果仍有乱码

1. 关闭 CMD 窗口
2. 用记事本打开 `.bat` 文件
3. 另存为 → 编码选择 **ANSI**
4. 保存后重试
