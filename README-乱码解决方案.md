# 解决乱码问题 - 完整方案

## 问题现状

在 Windows 上运行 `.bat` 文件时出现乱码：
```
'/b' 不是内部或外部命令
'鏃犳晠鐨勯€夐」' 不是内部或外部命令
```

**原因**：文件是 UTF-8 编码，Windows CMD 需要 GBK/ANSI 编码

---

## ✅ 解决方案（3 选 1）

### 方案 1：使用记事本转换（最简单，推荐）

1. 在 Windows 上找到文件
2. 右键 → 打开方式 → 记事本
3. 文件 → 另存为
4. 编码选择 **ANSI**
5. 保存

**需要转换的文件**：
- `软件/启动菜单.bat`
- `软件/软件启动工具/一键启动-Windows 系统.bat`
- `软件打包工具/一键打包-Windows.bat`

---

### 方案 2：使用 PowerShell 自动转换

在 Windows 上：

1. 打开 PowerShell（右键开始菜单 → Windows PowerShell）
2. 输入命令：

```powershell
cd "X:\你的路径\软件打包工具"
.\convert-files.ps1
```

3. 等待转换完成

---

### 方案 3：使用已创建的英文版（无需转换）

已创建英文版本，无需转换编码：
- `软件/run.bat`

直接双击即可运行（界面为英文）

---

## 🚀 转换后的使用

### 启动软件

1. 双击 `软件/启动菜单.bat`（转换编码后）
2. 输入数字选择软件
3. 完成

### 打包为 EXE

1. 转换 `软件打包工具/一键打包-Windows.bat` 的编码
2. 在 Windows 上双击运行
3. 等待自动打包完成
4. 获得 `.exe` 可执行文件

## 🔍 验证转换成功

转换成功后，双击 `.bat` 文件应该看到：

```
====================================
启动菜单
====================================

1 - 启动第一个软件
2 - 启动第二个软件
3 - 退出

选择 (1-3): _
```

如果看到乱码字符，说明转换失败，请重新操作。
